# ADR: Architecture Decision Records

## Template

```markdown
# 1. ADR の記事のタイトル
Date: 日付

## Status
proposed | rejected | accepted | deprecated | superseded by 記事リンク

## Context
この決定の背景やコンテキスト、決め手、制限等

## Decision
提案したい変更点、あるいは合意が交わされた変更点

## Other Options (任意)
他に検討した選択肢
なぜそれらを選ばなかったかの説明

## Consequences
この変更によって便利になること、不便になること
この変更によって生じるリスク、対処する必要があること
```
