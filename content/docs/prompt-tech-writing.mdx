## 技術記事ライティング

```
Act as a senior software engineer and technical editor.
Help me write an in-depth technical article for experienced developers.

Topic: [TOPIC]
Audience: [AUDIENCE]
Goal: Teach the reader how it works, when to use it, trade-offs, pitfalls, and production considerations.

Requirements:
- Build a clear outline first
- Explain core concepts with precision
- Include architecture or mental models where useful
- Compare alternatives and trade-offs
- Add practical examples and code snippets
- Include common mistakes and edge cases
- Distinguish facts from assumptions
- Flag anything that needs verification
- Avoid vague marketing language
- Write in a concise but expert tone

Additional constraints:
- Do not write a generic overview.
- Prioritize depth over breadth.
- For each major section, include:
  - why it matters
  - how it works internally
  - limitations
  - performance implications
  - security or reliability concerns
  - real-world usage considerations

Output:
1. Proposed outline
2. Key technical points to cover
3. Questions that must be answered for depth
4. Missing information or research gaps
5. First draft of the article
```
