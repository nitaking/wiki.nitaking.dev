title: Writing Prompts
description: Prompts for note-style reflective writing
---

技術記事とnote記事は全然違う。技術記事は情報の正確さと構造だけど、noteで刺さるのは具体性・手触り・迷い。

「AIっぽい」の問題は語彙じゃなく、書き手が見えないこと。解像度を上げるのが唯一の解決策。「緊張していた」より「ドアをノックする前にズボンで手のひらを3回拭った」。描写が精密なほど普遍的に響く。

Joan Didionの言葉が核心：「私は書くことで、自分が何を考えているか見つけている」。note記事は答えを伝えるんじゃなく、考えている過程を見せる文章。

AIの役割は聞き役まで。フェーズ1-2で素材を引き出して、繋ぐのは自分でやる。そのほうが自分が書いた感が残る。

---

## フェーズ1: 種を「場面」に戻す

```
私は記事の種を持っていますが、まだ抽象的です。
この種を書ける記事に育てるために、私に具体的な「場面」を
思い出させてください。

ルール:
- 一度に1つだけ質問する
- 私の答えが抽象的だったら「もっと具体的に。その瞬間、
  どこにいた?何が見えた?」と戻す
- 感情を聞く前に、行動と感覚(見た・聞いた・触った)を聞く
- 要約や解釈をしない。質問だけ

聞いてほしい順番:
1. この種が最初に芽吹いた瞬間は、いつ・どこで・何をしていた時か
2. その場に何があったか(視覚・音・匂い・手触り・温度)
3. その時、体は何をしたか(動作、姿勢、視線の先)
4. 誰か他の人はその場にいたか。その人は何をしていたか
5. その瞬間、頭の中で何と言葉にしていたか(内なる独り言)
6. その後、何が起きたか。いつ「これは書きたい」と思ったか
7. 今思い返して、あの時の自分に何と言いたいか

種:[ここに書きたいことを1-2文で書く]
```

## フェーズ2: 考えを割る(迷いを残す)

```
今の答えを踏まえて、以下を返してください。

1. 私の話には、矛盾しているように見える2つの気持ちや
   考えが隠れていないか。あるなら、それを指摘する
2. 私が無意識に「こう結論づけたい」と思っている答えは何か。
   それを指摘する
3. その結論を疑う視点を3つ挙げる
4. 結論を出さないまま、この矛盾や疑いをそのまま残して
   書いたらどういう記事になるか、骨格だけ示す

ここで結論を出さないでください。迷いを迷いのまま見せる骨格を。
```

noteで響く記事は結論が揺れている。「考えた、でも結局わからない、ただこう思った」という構造。

## フェーズ3: 書く時の制約(note向け)

```
以下の素材をもとに、noteに載せる記事のドラフトを書いて。

【この記事の性質】
- 情報を伝える記事ではない。考えている過程を見せる記事
- 結論を先に出さない。読者と一緒に迷う
- 一般論や教訓で締めない。個人的な具体のまま終わる勇気を持つ

【書き方の制約】
- 冒頭は場面から入る。具体的な時間・場所・動作から
- 「〜と言えるだろう」「〜ではないだろうか」「〜において」
  「昨今」「〜することが重要だ」は使わない
- 「〜ではなく、〜だ」の対比構文を多用しない
- 抽象語の直後には必ず具体例を置く
- 段落の長さを揃えない。1行段落と長い段落を混ぜる
- 読者に呼びかけない(「みなさん」「あなたも」を使わない)
- 箇条書きを使わない(このフォーマットの記事では)
- 「結論」「まとめ」という見出しを使わない

【声の指定】
私が普段書く文章は:[過去のnoteから2-3本貼るか、自分の文体を3文で説明する]

【素材】
[フェーズ1-2の答えを全部]

ドラフトは2000-3000字程度で。
見出しは2-3個、場面の描写で始める見出しに。
```
